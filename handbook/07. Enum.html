Enums

Enums

정적 상수를 정의하는 용도로 쓰고 아래와 같이 쓰면 된다.( 숫자 상수로 정의 )

enum Direction {
    Up = 1,
    Down,
    Left,
    Right
}
  • 숫자를 할당하면 그대로 할당
  • 아무것도 없다면 앞에것을 참조. 제일 앞에 것은 0
  • 괄호로 묶인 상수 열거표현식
  • 아무것도 없다면 다른 열거형에 대한 참조 가능( 근데 이거 참...)
  • +, - 등의 단항 연산자
  • +, -, *, /, %, <<, >>, >>>, &, |, ^ 등의 이진 연산( Nan, Infinity는 컴파일 오류)
enum FileAccess {
    // constant members
    None,
    Read    = 1 << 1,
    Write   = 1 << 2,
    ReadWrite  = Read | Write,
    // computed member
    G = "123".length
}

반대로 이름을 추출이 된다

enum Enum {
    A
}
let a = Enum.A;
let nameOfA = Enum[a]; // "A"

컴파일 한 결과

var Enum;
(function (Enum) {
    Enum[Enum["A"] = 0] = "A";
})(Enum || (Enum = {}));
var a = Enum.A;
var nameOfA = Enum[a]; // "A"

[번역] 생성 된 코드에서 enum은 forward (name -> value) 매핑과 reverse (value -> name) 매핑을 모두 저장하는 객체로 컴파일됩니다. 열거 형 멤버에 대한 참조는 항상 속성 액세스로 내보내지며 결코 인라인되지 않습니다. 많은 경우에 이것은 완벽하게 유효한 해결책입니다. 그러나 때로는 요구 사항이 더 엄격합니다. enum 값에 액세스 할 때 여분의 생성 된 코드 및 추가 indirection의 비용을 지불하지 않으려면 const 열거 형을 사용할 수 있습니다. const 열거 형은 enum 키워드 앞에 오는 const 한정자를 사용하여 정의됩니다.

const enum Enum {
    A = 1,
    B = A * 2
}

결과적으로 컴파일 시 enum에 대한 정의 값이 완전히 사라진다. 고로 const 역산은 불가능.

const enum Directions {
    Up,
    Down,
    Left,
    Right
}

let directions = [Directions.Up, Directions.Down, Directions.Left, Directions.Right]

var directions = [0 /* Up */, 1 /* Down */, 2 /* Left */, 3 /* Right */];

이건 결과적으로 결과가 다르다.

enum Directions {
    Up,
    Down,
    Left,
    Right
}

let directions = [Directions.Up, Directions.Down, Directions.Left, Directions.Right]


console.log(directions);
